# 头歌(Educoder)平台 C语言期末机考方案

> **课程**：C语言程序设计（信息类专业大一）  
> **方案制定日期**：2026-06-09  
> **方案状态**：待课程组研讨确认

---

## 一、考试基本信息

| 项目 | 内容 |
|------|------|
| 考试形式 | 头歌平台在线机考 |
| 考试时长 | **120分钟**（2小时）|
| 总分 | **100分** |
| 试题类型 | 选择题、程序阅读题、程序填空题、编程题 |
| 组卷方式 | **随机组卷**（每位考生试卷不同）|
| 评测方式 | 自动评测（选择题自动阅卷；编程题按测试用例自动评分）|

## 二、头歌平台功能对接方案

### 2.1 题型与头歌功能映射

| 考试题型 | 头歌支持情况 | 配置说明 |
|----------|--------------|----------|
| 单项选择题 | ✅ 原生支持 | 支持自动阅卷，可设置随机选项顺序 |
| 程序阅读题（写输出）| 通过**填空题**或**问答题**实现 | 填空：填输出结果；问答：手动阅卷（可选）|
| 程序填空题 | ✅ 通过**填空题**实现 | 挖空，填关键代码 |
| 程序改错题 | 通过**填空题**或**编程题**实现 | 填空：填改正后的代码行；编程：给错误代码让考生修正 |
| 编程题 | ✅ 原生支持（在线编程题）| 多测试用例自动评分，支持防硬编码 |

### 2.2 随机组卷配置

头歌「云组卷」功能支持以下随机策略：

```
策略说明：
1. 按题型比例抽题：
   - 选择题：从题库随机抽取 15 题
   - 程序阅读题：从题库随机抽取 4 题
   - 程序填空题：从题库随机抽取 3 题
   - 编程题：从题库随机抽取 2 题

2. 每位考生获得不同试卷（题目相同但顺序不同，或题目内容不同）

3. 难度控制：
   - 每题标注难度（基础/中等/较难）
   - 设置抽取比例：基础 60% + 中等 30% + 较难 10%
```

### 2.3 防作弊机制

| 机制 | 实现方式 |
|------|----------|
| 试题随机 | 每位考生题目顺序不同，部分题目内容不同 |
| 选项乱序 | 选择题选项随机排列 |
| 限时提交 | 单题可设时间限制（建议编程题单题限时30分钟）|
| 代码相似度检测 | 头歌平台内置代码查重（可选）|
| 全屏/防切换 | 头歌浏览器端可配置（需确认平台支持程度）|

## 三、试卷结构与分值设计

### 3.1 推荐试卷结构（100分，120分钟）

```
╔═══════════════════════════════════════════════════════════════╗
║  题型          题量   单题分值    总分    建议用时    难度分布      ║
╠═══════════════════════════════════════════════════════════════╣
║  一、单项选择题  15题  ×  2分  =  30分    20分钟    基础题为主      ║
║  二、程序阅读题   4题  ×  5分  =  20分    25分钟    基础+中等       ║
║  三、程序填空题   3题  ×  8分  =  24分    25分钟    中等为主        ║
║  四、编程题      2题  × 13分 =  26分    50分钟    中等+较难      ║
╚═══════════════════════════════════════════════════════════════╝
  合计：24题，100分，120分钟
```

### 3.2 各题型详细设计

#### 第一大题：单项选择题（15题 × 2分 = 30分）

**考察范围**（按章节权重）：

| 章节 | 题量 | 考察重点 |
|------|------|----------|
| C语言概述、开发环境 | 1题 | 程序结构、编译流程 |
| 基本数据类型、运算符 | 3题 | 数据类型范围、运算符优先级、类型转换 |
| 顺序/选择/循环结构 | 4题 | if/switch、for/while/do-while 语法与应用 |
| 数组 | 2题 | 数组定义、初始化、下标访问 |
| 函数 | 2题 | 函数定义、调用、参数传递（值传递）|
| 指针基础 | 2题 | 指针定义、& 和 * 运算符、指针作为函数参数 |
| 字符串 | 1题 | 字符串处理函数（strcmp/strcpy/strlen等）|

**题目特点**：
- 概念辨析为主，少量简单代码阅读
- 避免偏题怪题，聚焦核心概念
- 示例代码不超过10行

#### 第二大题：程序阅读题（4题 × 5分 = 20分）

**考察方式**：给出完整C程序，写出程序运行输出结果（填空形式）。

**难度梯次**：
- 第1题（基础）：顺序结构 + 简单运算，送分题
- 第2题（中等）：循环（for/while）+ 累加/累乘
- 第3题（中等）：数组遍历 + 最值/求和
- 第4题（较难）：递归函数 或 指针基础操作

**示例（待题库填充）**：
```c
#include <stdio.h>
int main() {
    int i, s = 0;
    for (i = 1; i <= 5; i++) {
        if (i % 2 == 1)
            s += i * i;
    }
    printf("s = %d\n", s);
    return 0;
}
// 问：程序输出结果是？
// 答：s = 35 （1² + 3² + 5² = 1 + 9 + 25 = 35）
```

#### 第三大题：程序填空题（3题 × 8分 = 24分）

**考察方式**：给出一个不完整程序，在指定位置填入缺失代码。

**题目设计原则**：
- 每题挖空 **2-3个关键位置**
- 空的类型：变量定义、循环条件、函数调用、return语句、指针操作等
- 不考察打字速度，聚焦逻辑理解

**典型场景**：
1. 用数组实现数据排序（冒泡/选择）
2. 字符串处理（连接/比较/反转）
3. 函数递归实现（阶乘/斐波那契）

#### 第四大题：编程题（2题 × 13分 = 26分）

**考察方式**：头歌在线编程题，多测试用例自动评分。

**题目难度**：
- 第1题（中等，13分）：单一知识点综合应用  
  示例：输入10个整数，用指针方式找出最大值和最小值
- 第2题（较难，13分）：多知识点综合  
  示例：用字符串和数组知识，实现简单的学号成绩管理（录入、查询、统计平均分）

**评分标准（头歌平台配置）**：

| 评分项 | 分值 | 说明 |
|--------|------|------|
| 测试用例通过 | 10分 | 通过所有隐藏测试用例 |
| 代码规范 | 1分 | 有适当注释、变量命名合理 |
| 时间效率 | 1分 | 无明显超时（O(n²)可接受，O(n!)不行）|
|  space效率 | 1分 | 无内存泄漏（大一可放宽）|

> **注**：代码规范分可由助教在头歌平台手动确认，或直接计入自动评分。

## 四、题库建设方案

### 4.1 题库规模要求

为保证随机组卷质量，题库需满足：

| 题型 | 试卷抽题量 | 题库建议量 | 冗余倍数 |
|------|--------------|--------------|----------|
| 单项选择题 | 15题 | **≥ 60题** | 4倍 |
| 程序阅读题 | 4题 | **≥ 20题** | 5倍 |
| 程序填空题 | 3题 | **≥ 15题** | 5倍 |
| 编程题 | 2题 | **≥ 10题** | 5倍 |
| **合计** | 24题/卷 | **≥ 105题** | - |

### 4.2 题目难度标注规范

每题需标注：
```
难度：基础 / 中等 / 较难
知识点：如"for循环"、"数组遍历"、"指针作为函数参数"
预计完成时间：如"3分钟"、"15分钟"
```

### 4.3 编程题测试用例设计原则

头歌编程题评分依赖测试用例，需设计：

| 测试类型 | 比例 | 说明 |
|----------|------|------|
| 基础样例 | 40% | 题目描述的示例输入，必过 |
| 边界测试 | 30% | 如空数组、最大值、最小值 |
| 随机测试 | 30% | 程序生成随机数据验证正确性 |

**防硬编码**：至少设计3组以上不同的测试用例，防止考生用 `if(input==x) printf(y)` 方式作弊。

## 五、考试实施流程

### 5.1 考前准备（课程组负责）

```
[第1周] 确认头歌课堂已创建，学生已完成注册
[第2周] 导入题库（详见第六章：题库文件格式）
[第2周] 在头歌平台配置考试：时长、开始/结束时间、随机组卷参数
[第3周] 安排模拟测试（见5.2节）
[第3周] 最终确认：试卷预览、评分规则、防作弊配置
```

### 5.2 模拟测试方案

**目的**：让学生熟悉头歌平台操作，同时检验题库和评分配置。

| 轮次 | 时间 | 内容 |
|--------|------|------|
| 第1轮 | 正式考试前2周 | 开放3道样题，学生熟悉平台界面和提交方式 |
| 第2轮 | 正式考试前1周 | 完整模拟考试（30分钟，5道题），检验随机组卷效果 |

**模拟考试后**：收集学生反馈，调整题库难度和平台配置。

### 5.3 正式考试日流程

```
考试前30分钟：  头歌平台开放"进入考试"按钮
考试开始：       系统自动计时，学生开始答题
考试结束前10分钟：系统弹窗提醒
考试结束：       系统自动提交，超时强行交卷
考试结束后：     平台自动生成成绩报告，教师可查阅各题得分分布
```

## 六、头歌平台题库导入格式

### 6.1 选择题导入格式（Excel模板）

头歌支持Excel批量导入，格式如下：

| 列名 | 说明 | 示例 |
|------|------|------|
| 题目类型 | 固定填"单选题" | 单选题 |
| 题目内容 | 题目文本（可含代码块）| 以下程序的输出是？`int a=5; printf("%d", a++);` |
| 选项A | | A. 4 |
| 选项B | | B. 5 |
| 选项C | | C. 6 |
| 选项D | | D. 不确定 |
| 正确答案 | 填A/B/C/D | B |
| 难度 | 基础/中等/较难 | 基础 |
| 知识点 | | 自增自减运算符 |

### 6.2 编程题在头歌的配置步骤

```
步骤1：在头歌课堂中创建"编程题"
步骤2：填写题目描述（支持Markdown格式）
步骤3：配置测试环境（GCC编译器，C99标准）
步骤4：添加测试用例（至少3组）
  - 测试名：test1_basic，输入：样例输入，期望输出：样例输出
  - 测试名：test2_boundary，输入：边界数据，期望输出：对应输出
  - 测试名：test3_random，输入：随机数据，期望输出：对应输出
步骤5：设置时间限制（如3000ms）和内存限制（如256MB）
步骤6：预览并发布到试卷
```

## 七、评分与成绩分析

### 7.1 自动评分说明

| 题型 | 评分方式 | 教师工作量 |
|------|----------|------------|
| 选择题 | 系统自动阅卷，零人工 | 无 |
| 程序阅读题 | 填空形式：系统自动阅卷 | 无（或少量）|
| 程序填空题 | 系统自动阅卷（精确匹配或正则匹配）| 无 |
| 编程题 | 系统按测试用例自动评分 | 无（代码规范可选人工）|

### 7.2 成绩报告分析（头歌平台提供）

考试结束后，头歌自动生成：
- **全班成绩分布图**（柱状图）
- **每题得分率**（识别教学薄弱点）
- **代码相似度报告**（可选，防作弊参考）
- **单个学生答题详情**（逐题得分、提交记录）
