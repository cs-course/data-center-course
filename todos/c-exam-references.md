## 参考资料一：清华大学《C语言程序设计》期末上机考试

**出处**：百度文库 - 清华大学《C语言程序设计》期末试卷  
**URL**：https://wenku.baidu.com/view/1a398b441db91a37f111f18583d049649b660ef5.html  
**访问时间**：2026-06-09

### 考试形式
- **上机考试**，考试时长 **3小时**
- 要求在C盘根目录建立以学号命名的子目录，所有程序存入该目录
- 例如：学号010000的同学，建立目录 `C:\010000`

### 题型与分值（参考）
| 题型 | 分值 | 说明 |
|------|------|------|
| 选择题 | 20分 | 基本概念、语法、程序理解 |
| 程序填空题 | 20分 | 补全代码 |
| 程序改错题 | 20分 | 找出并修正错误 |
| 编程题 | 40分 | 完整编写程序，通常3-4道 |

### 特色
- 注重**实际编程能力**考察，而非死记硬背
- 编程题覆盖：循环、数组、函数、字符串处理、文件操作
- 考试环境通常为 Turbo C / Dev-C++ / Code::Blocks

## 参考资料二：浙江大学《C语言程序设计》期末上机考试题库

**出处**：浙江大学课程攻略共享计划(ZJU-ICE) - GitHub  
**URL**：https://github.com/QSCech/zju-icecles/blob/master/程序设计基础/练习/C语言程序设计期末上机考试题库.pdf  
**访问时间**：2026-06-09

### 题型结构
| 题型 | 题量 | 分值 | 考察重点 |
|------|------|------|----------|
| 选择题 | 15题 | 30分 | 基础语法、数据类型、运算符 |
| 程序阅读题 | 5题 | 20分 | 阅读程序写结果 |
| 程序填空题 | 3题 | 15分 | 补全关键代码 |
| 编程题 | 3题 | 35分 | 完整编程实现 |

### 典型题目示例

**选择题示例**：
```c
// 以下叙述正确的是（）
// A）C语言的源程序不必通过编译就可以直接运行
// B）C语言中的每条可执行语句最终都将被转换成二进制的机器指令
// C）其他...
```

**程序阅读题示例**：
```c
// 读入2个整数lower和upper，输出华氏-摄氏温度转换表
// 华氏温度的取值范围是[lower, upper]，每次增加2°F
// 计算公式：c = 5 * (f - 32) / 9
```

**编程题示例**：
1. 写一个程序，输入一个整数，判断是否为素数
2. 输入10个整数，用冒泡排序法按从小到大排序
3. 编写函数，实现字符串反转

### 难度分布
- 基础题（送分题）：40%
- 中等题（区分度题）：40%
- 较难题（拔高题）：20%

## 参考资料三：Stanford CS107 系统编程期末笔试

**出处**：Stanford University, CS107: Computer Organization & Systems  
**URL**：https://web.stanford.edu/class/archive/cs/cs107/cs107.1202/exams/final/FinalExamQuestions.pdf  
**访问时间**：2026-06-09

> 注：CS107使用C语言教学，偏重系统编程，难度高于国内大一课程，但题型设计思路值得参考。

### 考试形式
- 笔试，时长 **2小时**
- 开卷/闭卷视学期而定
- 题型：简答题、代码分析题、编程题

### 题型特点（对国内机考的参考价值）
| 题型 | 参考用途 |
|------|----------|
| 代码阅读与预测输出 | 可直接移植为上机考试题 |
| 指针与内存分析 | 适合拔高题，考察深度理解 |
| 代码补全 | 适合填空题 |
| 小型编程题 | 适合上机编程题 |

### 对大一课程的借鉴
- 不要求内存管理、指针算术等难点
- 重点放在：基本语法、控制流、函数、数组、字符串
- 编程题以**算法实现**为核心，不追求语法偏题

## 参考资料四：头歌(Educoder)云组卷系统功能说明

**出处**：网易订阅 - 头歌实践教学平台  
**URL**：https://www.163.com/dy/article/GMOFHH5F0552CQFV.html  
**访问时间**：2026-06-09

### 平台支持的题型（共9种）
1. **选择题**（单选/多选）
2. **判断题**
3. **填空题**
4. **问答题**
5. **编程题**（自动评测，支持多测试用例）
6. **实训题**（完整项目/多文件）
7. **仿真题**
8. **Jupyter实训题**
9. **在线编程题**（类似OJ）

### 云组卷核心功能
| 功能 | 说明 |
|------|------|
| 试题批量导入 | 支持从Excel/Word批量导入试题 |
| 随机组卷 | 按难度/知识点/题型比例自动抽题 |
| 试题篮 | 像购物一样挑选题目，手动组卷 |
| 共建共享题库 | 教师可共享题目给其他教师 |
| 自动阅卷 | 选择/判断/填空自动评分；编程题按测试用例自动评测 |
| 防作弊 | 随机抽题使每位考生试卷不同 |

### 对期末机考的方案建议
- 使用**随机组卷**功能，设置3套以上等价试卷
- 编程题使用**多测试用例**评分，防止硬编码
- 每题设置**时间限制**（如单题最长30分钟）
- 结合**实训题**考察综合项目能力（可选，适合课程设计类考试）

### 常见题型与分值分布（综合多校）

| 题型 | 常见分值 | 备注 |
|------|----------|------|
| 单项选择题 | 20分（10题×2分） | 覆盖全课程 |
| 判断题 | 10分（10题×1分） | 概念辨析 |
| 程序阅读题（写输出）| 20分（4题×5分） | 重点考察 |

1. 数据类型与表达式（100%出现）
2. 三种基本结构（顺序/选择/循环）（100%出现）
3. 数组（含二维数组）（90%出现）
4. 函数（含递归基础）（85%出现）
5. 指针与字符串（70%出现）
6. 结构体与文件（50%出现，视课时而定）
7. 预处理与多文件（30%出现）

## 综合对比与建议

| 对比维度 | 清华大学 | 浙江大学 | Stanford CS107 | 头歌平台 |
|----------|----------|----------|-----------------|----------|
| 考试形式 | 纯上机 | 上机+笔试 | 笔试 | 在线机考 |
| 编程题占比 | 40% | 35% | 40% | 可自定义 |
| 自动评测 | 人工 | 人工 | 人工 | **全自动** |
| 随机性 | 无 | 无 | 无 | **支持随机组卷** |
| 适合大一 | ✅ | ✅ | ❌（偏难） | ✅ |

### 对本次机考方案的核心建议
1. **题型参考浙大**：选择+阅读+填空+编程，比例为 30:20:15:35
2. **随机组卷用头歌**：每个题型建题库，系统自动抽题生成不同试卷
3. **难度参考清华**：基础题60% + 中等题30% + 难题10%
4. **编程题仿CS107**：重算法逻辑，不考偏门语法
5. **防作弊**：随机组卷 + 每题多个等价测试用例 + 限时提交
